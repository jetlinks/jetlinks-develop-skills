# JetLinks Web Business UI Example Rules

本文件用于规范“参考类似业务设计”时该怎么借鉴、哪些情况不能借。

## 参考优先级

1. 当前模块相邻页面
2. 同业务域页面
3. 相似业务案例
4. Ant Design 官方交互模式
5. 泛化设计灵感

优先级越低，越不能直接借交互结构。

## 允许借鉴的内容

- 信息分区方式
- 操作组织方式
- 状态反馈方式
- 空 / 错 / 载态处理
- 详情与列表的切换关系

## 禁止直接照搬的内容

- 完全不同业务的指标体系
- 与当前对象模型不一致的流程步骤
- 与当前任务无关的首屏统计卡
- 另一个场景里的快捷操作布局
- 另一个业务中的告警层级、审批链、分栏逻辑

## 正例

### 正例 1：设备异常处置页参考告警处置台

可借：
- 优先级排序
- 待处理区 + 详情区组合
- 状态高亮方式

原因：
- 两者都强调“快速发现问题并采取动作”

### 正例 2：单对象配置详情页参考单对象详情页

可借：
- 顶部摘要
- Tabs 分区
- 固定操作区

原因：
- 两者都围绕单个对象持续查看与操作

## 反例

### 反例 1：设备配置页照搬经营分析大盘

不应借：
- 大量经营 KPI 卡
- 趋势图墙
- 排行榜

原因：
- 当前任务是配置与执行，不是经营判断

### 反例 2：告警处置页照搬订单管理表

不应借：
- 纯筛选 + 表格 + 弹窗结构

原因：
- 告警处置依赖状态理解、优先级和连续动作，不是简单批量 CRUD

### 反例 3：详情页默认加 4 个统计卡

不应借：
- “总数 / 今日新增 / 本周处理 / 完成率” 这类模板化数字

原因：
- 如果数字不驱动判断和动作，只是在浪费首屏空间

## 借鉴说明模板

当你决定参考某个案例时，先明确说明：

- 参考对象：
- 相似点：
- 只借鉴什么：
- 明确不借什么：

如果这四项说不清楚，就不要借。
