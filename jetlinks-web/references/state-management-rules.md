# JetLinks Web State Management Rules

本文件用于选择状态承载方式，确保状态边界清晰、请求链路稳定。

## 适用范围

- 判断状态放组件本地、Pinia store、路由参数还是服务端缓存
- 页面初始化、权限驱动展示、跨页面共享状态

## 状态分层原则

- 组件本地状态：弹窗开关、临时输入、短生命周期 loading
- 全局 store 状态：跨页面复用、路由守卫依赖、全局配置/菜单/用户信息
- 路由状态：可分享筛选条件、回跳上下文
- 服务端状态：通过 store action 统一拉取与落库

## 推荐工作流

1. 先判断状态生命周期和作用域。
2. 短生命周期状态优先放组件内，不上全局 store。
3. 全局共享数据通过 store action 拉取，避免页面重复请求。
4. 在组件中使用 `storeToRefs` 消费响应式 state，动作方法直接从 store 取。
5. 页面初始化先检查缓存，再触发请求。

## 约束

- 不要把单页临时状态塞进全局 store。
- 不要在多个页面复制同一请求逻辑。
- 不要绕过现有 store 契约在页面散落全局副作用。

## 自检清单

- 状态边界是否合理（本地 vs 全局）。
- 是否避免了重复请求与无效刷新。
- 是否使用可维护的 store 消费方式。
- 权限、菜单、系统配置等全局状态是否复用既有 store。
