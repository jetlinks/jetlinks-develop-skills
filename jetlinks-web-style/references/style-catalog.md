# 页面风格目录

## 1. 标准管理表格页

### 适用场景

- 字段稳定、列结构清晰
- 主要任务是筛选、查看、批量操作、简单新增编辑
- 用户对数据密度要求高，但不需要复杂工作台式导航

### 结构骨架

- 标题 / 说明
- 顶部搜索区
- 表格工具栏
- 表格
- 常规分页

### 不适用

- 表格字段过多，真正重点是先定位记录再看摘要
- 需要左侧快捷筛选、已保存搜索、表达式筛选协同
- 页面主要价值不在表格本身

## 2. 筛选工作台式台账页

### 适用场景

- 用户第一任务是“高频筛选后快速定位记录”
- 记录项较复杂，需要列表项表达业务重点，而不是纯列平铺
- 筛选比表格更重要，且经常需要组合条件、快捷筛选、已保存搜索
- 适合资产、告警记录、工单记录、订单台账、账号资产等页面

### 结构骨架

- 标题 / 一句面向用户的说明
- 顶部条件筛选工具条（通常以表达式筛选为主）
- 左侧快捷筛选侧栏（状态、区域、已保存搜索等）
- 主内容台账列表 / 紧凑列表
- 轻量分页（总数 + 上一页 / 下一页）

### 固定锚点

- 表达式筛选输入是主入口
- 快捷筛选与主筛选联动，而不是彼此独立
- 列表更像“记录摘要”，不是传统表格列平铺
- 工具操作收敛，不堆满按钮

### 不适用

- 记录字段很少，普通表格已足够
- 页面主要目标是单对象编辑或配置
- 没有持续筛选需求，左侧侧栏会变成噪音

## 3. 主从详情工作区

### 适用场景

- 用户一边浏览列表，一边查看或编辑单个对象详情
- 左侧对象导航 / 列表，右侧详情成为主工作区
- 适合设备分组、项目成员、模板配置、规则节点等需要上下文切换的场景

### 结构骨架

- 标题 / 说明
- 左侧树、列表或分组导航
- 右侧详情 / 编辑 / 子配置区域
- 顶部或右侧局部操作

### 不适用

- 页面第一任务是批量筛选记录
- 右侧详情很轻，不值得做工作区

## 4. 配置表单页

### 适用场景

- 页面目标是完成一次配置、录入或编辑
- 用户主要操作一个对象，而不是一批对象
- 表单、分组、校验和提交流程是主线

### 结构骨架

- 标题 / 说明
- 分组表单或步骤表单
- 固定提交区

### 不适用

- 页面主要任务是检索记录
- 需要持续切换多条对象

## 5. 概览 / 工作台页

### 适用场景

- 页面重点是监控、分析、总览、异常态势
- 卡片、图表、趋势、排行都有明确数据来源和决策用途

### 结构骨架

- 概览指标
- 趋势 / 分析区
- 关键异常或待办区
- 必要的 drill-down 入口

### 不适用

- 没有真实统计来源
- 只是想让页面“看起来丰富”

## 风格选择时的推荐措辞

推荐固定用下面格式：

1. 推荐：`筛选工作台式台账页`
   - 适合：高频筛选后定位复杂记录
   - 结构：顶部条件筛选 + 左侧快捷筛选 + 台账列表
2. 备选：`标准管理表格页`
   - 适合：字段稳定、批量操作为主
3. 备选：`主从详情工作区`
   - 适合：列表浏览与单对象深度编辑并行

如果用户没有明确偏好，给出默认项并直接说明原因。
