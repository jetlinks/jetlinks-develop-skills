# 页面风格目录

## 1. 标准管理表格页

### 适用场景

- 字段稳定、列结构清晰
- 主要任务是筛选、查看、批量操作、简单新增编辑
- 用户对数据密度要求高，但不需要复杂工作台式导航

### 结构骨架

- 标题 / 说明
- 顶部搜索区
- 表格工具栏
- 表格
- 常规分页

### 不适用

- 表格字段过多，真正重点是先定位记录再看摘要
- 需要左侧快捷筛选、已保存搜索、表达式筛选协同
- 页面主要价值不在表格本身

## 2. 筛选工作台式台账页

### 适用场景

- 用户第一任务是“高频筛选后快速定位记录”
- 记录项较复杂，需要列表项表达业务重点，而不是纯列平铺
- 筛选比表格更重要，且经常需要组合条件、快捷筛选、已保存搜索
- 适合资产、告警记录、工单记录、订单台账、账号资产等页面

### 结构骨架

- 标题 / 一句面向用户的说明
- 顶部条件筛选工具条（通常以表达式筛选为主）
- 左侧快捷筛选侧栏（状态、区域、已保存搜索等）
- 主内容台账列表 / 紧凑列表
- 轻量分页（总数 + 上一页 / 下一页）

### 固定锚点

- 表达式筛选输入是主入口
- 快捷筛选与主筛选联动，而不是彼此独立
- 列表更像“记录摘要”，不是传统表格列平铺
- 工具操作收敛，不堆满按钮

### 不适用

- 记录字段很少，普通表格已足够
- 页面主要目标是单对象编辑或配置
- 没有持续筛选需求，左侧侧栏会变成噪音

## 3. 主从详情工作区

### 适用场景

- 用户一边浏览列表，一边查看或编辑单个对象详情
- 左侧对象导航 / 列表，右侧详情成为主工作区
- 适合设备分组、项目成员、模板配置、规则节点等需要上下文切换的场景

### 结构骨架

- 标题 / 说明
- 左侧树、列表或分组导航
- 右侧详情 / 编辑 / 子配置区域
- 顶部或右侧局部操作

### 不适用

- 页面第一任务是批量筛选记录
- 右侧详情很轻，不值得做工作区

## 4. 配置表单页

### 适用场景

- 页面目标是完成一次配置、录入或编辑
- 用户主要操作一个对象，而不是一批对象
- 表单、分组、校验和提交流程是主线

### 结构骨架

- 标题 / 说明
- 分组表单或步骤表单
- 固定提交区

### 不适用

- 页面主要任务是检索记录
- 需要持续切换多条对象

## 5. 概览 / 工作台页

### 适用场景

- 页面重点是监控、分析、总览、异常态势
- 卡片、图表、趋势、排行都有明确数据来源和决策用途

### 结构骨架

- 概览指标
- 趋势 / 分析区
- 关键异常或待办区
- 必要的 drill-down 入口

### 不适用

- 没有真实统计来源
- 只是想让页面“看起来丰富”

## 6. 分步流 / 向导页

### 适用场景

- 用户必须按阶段完成配置、开通、发布、审批或联动流程
- 阶段之间存在顺序、依赖、校验或回退关系
- 单页平铺会让用户难以判断当前进度和下一步动作

### 结构骨架

- 标题 / 当前流程说明
- Steps / 分段导航
- 当前阶段表单、配置或确认区
- 阶段结果、校验提示和下一步动作

### 不适用

- 各区块彼此独立，没有强顺序关系
- 用户主要任务是检索或批量处理对象

## 7. 时间线 / 记录分析页

### 适用场景

- 用户需要理解事件演进、操作历史或状态变化
- 决策依赖“先发生什么、后发生什么、为何变化”
- 适合日志、告警流、审批记录、设备事件、任务执行轨迹等页面

### 结构骨架

- 时间或条件过滤区
- Timeline / 日志流 / 记录流
- 关键节点高亮
- 节点详情、关联对象和后续动作

### 不适用

- 记录之间没有时序关系
- 普通表格已能完成主要检索和处理任务

## 8. 树表 / 分组管理页

### 适用场景

- 对象天然具有树、分组、目录、区域或分类层级
- 用户先通过层级定位范围，再管理右侧列表或详情
- 适合组织、区域、分类、设备分组、菜单权限等页面

### 结构骨架

- 左侧树 / 分组导航
- 右侧搜索区或局部工具栏
- 右侧表格、卡片列表或详情配置
- 层级新增、移动、重命名等局部操作

### 不适用

- 层级只是一个普通筛选条件，不需要长期占据左侧空间
- 用户主要按复杂条件筛选，而不是按固定层级定位

## 风格选择时的推荐措辞

推荐固定用下面格式：

1. 推荐：`筛选工作台式台账页`
   - 适合：高频筛选后定位复杂记录
   - 结构：顶部条件筛选 + 左侧快捷筛选 + 台账列表
2. 备选：`标准管理表格页`
   - 适合：字段稳定、批量操作为主
3. 备选：`主从详情工作区`
   - 适合：列表浏览与单对象深度编辑并行
4. 备选：`分步流 / 向导页`
   - 适合：阶段顺序与校验关系明确的配置流程
5. 备选：`时间线 / 记录分析页`
   - 适合：按事件演进理解状态变化
6. 备选：`树表 / 分组管理页`
   - 适合：先按层级定位，再管理列表或详情

如果用户没有明确偏好，给出默认项并直接说明原因。
