# JetLinks 国际化规则

本文件用于在 JetLinks 系脚手架中补充用户可见文案的国际化，但前提是当前模块已经存在或明显采用了 i18n 约定。

## 第一原则：先确认当前模块是否真的使用 i18n

检查：

- 是否存在 `src/main/resources/i18n/...`
- 是否同时存在中英文资源
- 相邻模块是否对权限、动作、枚举、字段做了本地化

如果当前脚手架没有这套约定，不要为了“形式完整”而臆造一套新的资源路径。

## 何时需要补 i18n

- 新增枚举且前端会直接展示其文本
- 新增实体字段且当前模块已有字段国际化惯例
- 新增 `@Resource` 权限 ID 且当前模块已有权限国际化
- 新增 `@ResourceAction` 或等价动作 ID 且当前模块已有动作国际化
- 新增用户可见错误或提示消息

## 常见键模式

以下是常见模式，但是否采用以当前模块现状为准：

- 枚举：`完整包名.枚举类名.枚举值`
- 实体字段：`完整包名.实体类名.字段名`
- 权限：`hswebframework.web.system.permission.{权限ID}`
- 动作：`hswebframework.web.system.action.{动作ID}`
- 错误：`error.{key}`
- 提示：`message.{key}`

## 编写规则

1. 键名和代码中的标识保持一一对应
2. 若模块已有 `messages_zh.properties` 和 `messages_en.properties`，尽量同步补齐
3. 如果相邻模块只维护一种语言，跟随现有惯例
4. 不要为内部日志、调试信息补国际化

## 注意事项

- 当前工作区可能存在“已有权限注解但未完全补 i18n”的历史情况
- 因此提示词不能写成绝对强制，而应要求智能体先判断模块习惯，再补一致的内容

## 自检清单

- 当前模块是否已有 i18n 目录
- 这类键在相邻模块中是否已被使用
- 新增的权限、动作、枚举、字段是否与代码标识一致
- 多语言文件是否保持同步
